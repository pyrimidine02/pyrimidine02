# Welcome

## 간략 소개
- 백엔드 개발과 인프라/운영에 관심이 있습니다.
- 위치 기반(LBS) 도메인에서 인증/권한, 콘텐츠/지리 데이터 처리, 운영 품질(관측성·레이트리밋·오류 표준화)을 다룹니다.
- OpenAPI-first, 계층 분리, RBAC/감사 로그 같은 운영 친화적 원칙을 선호합니다.

## 취미

- 성우/서브컬처 덕질
- 일본 공연 원정
- 기타
- DTM/작곡 (Logic Pro)
- 사진/카메라

## 프로젝트

### Girls Band Tabi
애니메이션·걸즈밴드 프로젝트 팬을 위한 위치 기반 성지순례(LBS) 플랫폼 백엔드.

- 목표
  - 프로젝트/유닛/장소/라이브/뉴스/방문 인증/이미지 업로드 등 팬 활동 데이터를 단일 API로 제공
- 아키텍처/구성
  - Kotlin + Spring Boot 기반 API 서버
  - Spring Modulith 기반 이벤트 드리븐 모듈 구조 (도메인 분리)
- 주요 기능
  - 인증/권한: JWT + OAuth2(소셜) + 프로젝트 단위 RBAC, 감사 로그
  - 위치/지리: PostgreSQL + PostGIS + Hibernate Spatial 기반 위치 검증(반경/다각형) 및 근접성 계산
  - 운영 품질: Redis 기반 캐시/레이트리밋, 서킷 브레이커 기반 장애 격리
  - 스토리지: Cloudflare R2(S3 호환) 업로드 플로우(서명 URL)
  - 관측성: Actuator/Micrometer 지표, Prometheus 수집, 로그 구조화
